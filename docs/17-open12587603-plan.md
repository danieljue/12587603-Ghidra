# Open12587603 — Rebuild Plan

> **Version 2.0 — Complete rebuild based on verified disassembly analysis**
> No more guessing. Every address verified. Every hook traced. Every routine designed before coding.

## What We Learned From V1

V1 of the patcher was wrong because:
1. We placed calibration tables without code that reads them
2. We guessed RAM addresses that don't exist (0xFFFFAD32 ≠ VSS)
3. We added "features" that were just dead bytes in flash
4. We didn't know the factory already has launch control logic
5. We didn't know Boost OS repurposes EGR hardware

## What We Now Know (Verified)

### Hardware
- VSS: 0xFFFFAEC0 (raw frequency)
- RPM: 0xFFFFA562 (raw, /5.12 = engineering)
- MAP: 0xFFFFADB6 (scaled internal format)
- Clutch switch: 0xFFFFA3AB (processed state)
- Gear position: 0xFFFFA3AF (byte, >5 = in-gear)
- Fuel cut status: 0xFFFFA93C
- Boost control output: C2 Green Pin 3 (EGR EVAP vent solenoid, re-purposed)
- Wideband input: C1 Blue Pin 55 (EGR pintle position ADC, 0-5V)

### Functions
- Table interpolation: sub_16D6 (0x0016D6) — used by ALL table lookups
- RPM limiter: sub_30368 (0x030368) — hook at +0x36 and +0x90
- VE lookup: sub_79B10 (0x079B10) — 3 hook points
- Factory launch spark: sub_3BDC4 (0x03BDC4) — already exists!
- Factory FFS spark: sub_2FC1E (0x02FC1E) — already exists!
- Clutch DFCO: sub_30566 (0x030566)

### Boost OS Actual Changes (from binary diff)
- 6 JSR hooks into OS code
- ~9.2 KB custom code in OS5 gap (0x0D0000)
- ~7.6 KB data in OS7 (0x0E0000)  
- VE expansion: redirects VE table pointer when MAP > 100 kPa
- EGR repurposed: EGR solenoid = boost control, EGR ADC = wideband input

## Rebuild Architecture

### Phase 1: VE Expansion (The Only Required New Code)

This is the ONLY feature that requires new 68k code. Everything else either:
- Already exists in the factory OS (launch control, FFS)
- Or only needs calibration changes

```
Hook: 3 JSR patches in sub_79B10 (at 0x079B92, 0x07A176, 0x07A95E)
      replace: lea (0x00008442).l, a0
      with:    jsr VE_EXPANSION_ROUTINE

VE_EXPANSION_ROUTINE (at 0x0D0000):
  1. Read MAP from FFFFADB6
  2. If MAP < boost_threshold: a0 = &factory_VE (0x8442), rts
  3. If MAP >= boost_threshold: a0 = &expanded_VE (0x0E0100), rts
  4. NOTE: must also adjust d2 (stride) if expanded table has different dimensions
```

### Phase 2: Enable Factory Features (Calibration-Only)

These require NO new 68k code — just modifying existing calibrations:

| Feature | What to Change | Address(es) |
|---------|---------------|-------------|
| Launch Control | Enable launch spark, set RPM thresholds | SPARK_ADVANCE calibrations |
| Flat Foot Shift | Enable FFS spark blend | SPARK_ADVANCE_KV_FFS_SPARK_BLEND_FACTOR |
| 2-Step Rev Limit | Lower fuel-cut thresholds in P/N | KE_PN_ENGINE_OVERSPEED_HIGH/LOW |
| Spark Cut Limiter | Set spark cut RPM thresholds | SPARK_ADVANCE calibrations |

### Phase 3: Boost-Specific Features (New Tables + Some New Code)

| Feature | New Tables | New Code | Complexity |
|---------|-----------|----------|------------|
| Boost Spark Adder | 17×17 table at 0x0E0200 | Hook into spark calc chain | Medium |
| Boost VE Table | 33×17 at 0x0E0100 | VE expansion routine (Phase 1) | Medium |
| Open Loop Boost Control | Target boost vs RPM, PID gains | EGR solenoid PWM control | Hard |
| Wideband Closed Loop | AFR target table | ADC reader + PID + fuel trim hook | Hard |
| Lean Boostcut | AFR/MAP thresholds | MAP check + fuel cut injection | Medium |
| Overboost Protection | MAP threshold scalar | MAP check + spark cut injection | Medium |

## Design Principles for V2

1. **No dead code.** Every byte placed in flash must be read by a 68k routine.
2. **Verified addresses only.** No guessed RAM addresses — every address traced from disassembly.
3. **One feature at a time.** VE expansion first. Everything else builds on it.
4. **Documented hooks.** Every JSR patch must include: original bytes, patched bytes, rationale, data flow diagram.
5. **AI-consumable docs.** Structured markdown so AI agents can load and understand the entire project.
