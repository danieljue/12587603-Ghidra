# Open12587603 — Project Charter

> **Goal**: Build a fully open-source custom OS patcher for P59 OS 12587603 that replicates and exceeds Boost OS features.
> **License**: MIT — anyone can use, modify, distribute.
> **Tagline**: "Boost OS features, open-source license, Ghidra-powered."

---

## Why This Matters

Boost OS is excellent but closed-source and Patreon-gated for V4+. The community deserves an open alternative that:

1. **Anyone can audit** — every patch is documented with exact byte offsets and rationale
2. **Anyone can modify** — add your own features without reverse-engineering from scratch
3. **Anyone can contribute** — pull requests welcome
4. **Is Ghidra-powered** — all patch addresses are cross-referenced against the annotated disassembly
5. **Is AI-friendly** — designed for programmatic use via Hermes/ghidra-headless-mcp

## Immediate Next Steps

### Step 1: Get Boost OS Files
If you can download the V3 ZIP from pcmhacking.net (free) or the V4 from Patreon, we can:
- Binary-diff a stock bin vs Boost OS patched bin → extract exact patches
- Analyze the Boost OS XDF → understand new table layouts
- Reverse-engineer the patcher → understand the patching algorithm

### Step 2: Build First Feature — Launch Control
Even without Boost OS files, we know enough to build launch control:
1. Find VSS RAM address in disassembly
2. Find clutch switch input bit
3. Write 68k subroutine: check VSS==0 AND clutch → use launch RPM limit
4. Inject at 0xA0000 (OS5 free space)
5. Patch `sub_30368` to JSR to our routine before threshold selection
6. Add launch RPM calibration table at 0xC0000 (OS6)
7. Generate XDF entry

### Step 3: Build the Patcher
```python
open12587603.py \
  --input stock_12587603.bin \
  --output custom_12587603.bin \
  --features launch_control,expanded_ve,boost_spark \
  --launch-rpm 4500 \
  --map-sensor 2bar \
  --generate-xdf
```

### Step 4: Document Everything
Every patch gets a markdown file explaining:
- What it does (user perspective)
- How it works (technical perspective)
- Which addresses are modified and why
- Cross-reference to Ghidra disassembly
